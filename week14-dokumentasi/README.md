# Minggu 14 — Dokumentasi Perancangan

**Terkait CPMK:** 31.6

## Tujuan

Menyusun Software Design Document yang mengintegrasikan requirement, seluruh
model UML, implementasi, traceability, keamanan, dan transparansi AI.

## Cara Menjalankan

```bash
node generate-docs.js
```

Hasil: `SDD-generated.md`.

## Pemeriksaan Wajib

1. Setiap diagram konsisten dengan requirement dan kode.
2. Tidak ada elemen hasil AI tanpa dasar requirement.
3. Matriks traceability dilengkapi.
4. Risiko keamanan, privasi, prompt injection, dan AI Agent dijelaskan.
5. Catatan penggunaan AI diisi berdasarkan aktivitas nyata.
6. Draf otomatis tidak boleh diserahkan tanpa penyuntingan manusia.
