# Minggu 3 — Analisis Kebutuhan Sistem

**Terkait CPMK:** 21.3

## Tujuan
Berlatih menerjemahkan kebutuhan pengguna (requirement) menjadi user story,
lalu mengidentifikasi entitas/class awal dari kebutuhan tersebut.

## Konsep Kunci
- Elisitasi kebutuhan (fungsional vs non-fungsional)
- Format user story: *Sebagai [peran], saya ingin [aksi], agar [tujuan]*
- Elisitasi kebutuhan berbantuan AI: coba minta AI assistant membuatkan draf
  user story dari deskripsi kasus singkat, lalu **bandingkan** dengan versi
  manualmu — apakah ada kebutuhan yang terlewat atau dikarang AI (halusinasi)?

## Cara Menjalankan
```bash
node index.js
```

## Berkas
- `requirements.md` — daftar kebutuhan & user story (lengkapi sesuai tugas)
- `index.js` — skeleton class hasil identifikasi entitas dari kebutuhan

## Tugas Latihan
1. Lengkapi `requirements.md` sesuai instruksi di dalamnya.
2. Jika ditemukan entitas baru, tambahkan skeleton class-nya di `index.js`.
3. Gunakan Kerangka Evaluasi Kritis (lihat dokumen RPS) untuk menilai hasil
   AI generation-mu pada poin "Elisitasi kebutuhan berbantuan AI" di atas.
