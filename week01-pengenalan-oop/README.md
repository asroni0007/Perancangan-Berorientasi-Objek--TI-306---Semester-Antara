# Minggu 1 — Pengenalan Perancangan Berorientasi Objek

**Terkait CPMK:** 21.1

## Tujuan

1. Membandingkan paradigma prosedural dan berorientasi objek.
2. Mengenal tren pengembangan perangkat lunak modern: AI-assisted development,
   cloud-native, dan low-code/no-code.
3. Menilai kapan OOP memberi manfaat dan kapan menambah kompleksitas.

## Konsep Kunci

- Prosedural: data dan fungsi cenderung dipisahkan.
- OOP: data dan perilaku dibungkus dalam class/object.
- Encapsulation menjaga aturan bisnis agar tidak diubah sembarangan.
- AI-assisted development membantu drafting/review, tetapi membutuhkan verifikasi.
- Cloud-native menekankan deployment, skalabilitas, observability, dan otomasi.
- Low-code/no-code mempercepat pembuatan solusi tertentu, tetapi memiliki batas
  fleksibilitas, kontrol, dan maintainability.

## Cara Menjalankan

```bash
node index.js
```

## Aktivitas

1. Jalankan dua pendekatan pada `index.js`.
2. Tambahkan aturan: buku kategori `Referensi` memiliki tarif denda dua kali lipat.
3. Implementasikan aturan pada versi prosedural dan OOP.
4. Bandingkan jumlah bagian kode yang harus berubah.
